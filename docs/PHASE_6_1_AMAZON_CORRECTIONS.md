# Phase 6.1 Amazon Corrections

This focused repair adds real Amazon template header discovery, exact stored file/worksheet selection, supporting-report preservation, direct immutable snapshot scanning, deterministic catalog authority, and compact marking-card image fallback.

No mobile, Expo, Android, EngravingBrain, Worker Agent, inventory, or ERP work is included.

Catalog authority is deterministic by safe file/table/row provenance. All Listings controls identity/status; catalog reports enrich product display fields. Blank cells never erase nonblank values, and conflicting nonblank catalog values create `CATALOG_FIELD_CONFLICT` warnings without storing raw rows or managed paths.
